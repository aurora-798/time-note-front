需求分析
我本身有想要过培养写日记的习惯，但是比较难培养，人的记忆总是会忘记之前部分发生过的事情，于是就有了一个需求：能不能不写日记，让电脑自动帮我记录今天所发生的事情，只需要通过语音或者文字的形式告诉电脑，让电脑自动生成当天的日记即可。
核心功能
1. 用户通过语音/文字形式告诉AI今天发生的事情，AI 自动总结并生成日记。
必要功能
管理员
1. 管理员登陆
2. 用户管理（新增、编辑、查询、删除、status、isvip）
3. 日记管理（新增、编辑、查询、删除）
4. Echats图表：总 Token 消耗、单用户消耗 Token 数、总注册用户数量、男女比例等
用户
1. 登陆注册功能
2. 个人中心（头像、昵称、风格、年龄、性别等）
3. 日记编辑、保存、删除、查看、导出(单篇/批量)；查看日记列表等；图片/视频上传功能(类似豆包形式，先上传图片/视频，再通过语音/文字形式描述图片/视频)；
   技术选型
   后端：Spring Boot3 + Spring Security + LangChain4j
   数据库：MySQL + Redis
   前端：Vue3

目前我后端做完了一部分了，后端接口文档：docs/后端接口文档.md
